1. Введение

Данная программа написана в среде программирования Borland Delphi 6 на языке Object Pascal. Проект состоит из двух основных частей – самой программы (7 основных модулей) и библиотеки компонентов (7 дополнительных модулей).

Для работы необходим компьютер с установленной операционной системой Microsoft Windows 95/98/NT/ME/2000/XP.

При написании программы были использованы стандартные компоненты библиотеки Borland VCL (Visual component library) и на их основе были построены новые компоненты специально для программы.

Программа занимает не много места на диске (621 Кб), не требовательна к системным ресурсам, а также для своей работы не требует каких-либо специальных библиотек (как например, программы, написанные на Visual Basic). Все измененные параметры настройки сохраняются.

Программа предназначена для изучения отдельных вопросов алгебры логики, а более конкретно для построения минимальной дизъюнктивной нормальной формы. Кроме того, программа обеспечивает построение логической схемы для сложной логической функции.

Основное ее назначение – обучающая программа.

 

Раздел «элементы алгебры логики» в курсе информатики школ, техникумов и вузов является достаточно сложным для изучения и требует при его рассмотрении большей наглядности. Представляемая программа отчасти решает эту проблему.

2. Работа программы происходит следующим образом.

1) На первом этапе пользователь задает текстовую логическую функцию с использованием следующих обозначений:

"+"  –  дизъюнкция;

"*" или "&"  –  конъюнкция;

"Ø"  – отрицание ("-" на клавиатуре);

">" – импликация;

"=" – эквиваленция.

Кроме того, логическая функция может содержать скобки.  Максимальное количество логических переменных – 6.

Для введенных логических функций проводится проверка на корректность. Если введена некорректная функция, выводится сообщение об ошибке.

2) На втором этапе для введенной логической функции программа составляет таблицу истинности.

Для этого был реализован алгоритм "обратных приоритетов": функция рекурсивно разбивается на части.

Рекурсивная функция (DivFunc) состоит из нескольких основных частей (см. Приложение 2):

1)     Определение операции с наименьшим приоритетом (с учетом скобок)

2)     Выделение в текстовой функции в зависимости от операции одной или двух частей, например, для функции  операцией с наименьшим приоритетом будет сложение и соответственно функция будет разделена на две части: и C.

3)     Рекурсивный вызов функции если полученные части не отдельные переменные

При этом все промежуточные результаты заносятся в специальные динамические структуры. В результате получается "дерево", ветвями которого являются отдельные логические операции. На основе полученного дерева производится расчет значений функции при всех возможных значениях переменных, и составляется таблица истинности. Это делается с помощью функции EvaluateFunc (см. Приложение 4). Сначала производится расчет для начальной ветви динамического дерева :

1)     Определяется тип логической операции

2)     Определяются составные части логической функции. Если это другие логические функции, то производится рекурсивный расчет для этой функции иначе определяется значение логической переменной

3)     В зависимости от логической функции рассчитывается значение функции при данных значениях переменных

Все эти операции производятся для всех возможных значений переменных

3) На третьем этапе по таблице истинности составляется любая форма – по выбору пользователя – СДНФ (GetSDNF; совершенная дизъюнктивная нормальная форма), или СКНФ (GetSCNF; совершенная конъюнктивная нормальная форма), или МДНФ –  минимальная дизъюнктивная нормальная форма. Создание МДНФ происходит поэтапно по методу Квайна-МакКласски. Это позволяет прослеживать преобразования логической функции на экране компьютера.

Метод состоит из четырех шагов:

·        Представим наборы, на которых функция истинна, в виде двоичных эквивалентов

·        Упорядочим двоичные эквиваленты по ярусам и проведем склейку наборов в соседних ярусах, получая максимальные интервалы до тех пор, пока это возможно; помечаем каждый набор, участвовавший в склейке. Склеиваются только те наборы или интервалы, различие в которых заключается в значении одного разряда:

001                   001-               -01-

000                   101-               -11-

·        Поглощение повторяющихся наборов

·        Составление функции в виде бинарного набора

·        Составление функции в текстовом виде

Данный алгоритм выполняет функция GetMDNF (см. Приложение 4)

4) На четвертом этапе программа производит построение логической схемы на основе построенного нового дерева. При необходимости построение логической схемы может производиться для любой введенной пользователем функции.

Программа имеет простой и удобный редактор, предназначенный для создания и редактирования логических схем. Для работы редактора логических схем была построена объектная модель, содержащая компоненты:

1)     TlogEdit – наследник TEdit – обрабатывает нажатия клавиш:

Автоматически заменяет символы

"*", "&" на "•" - знак дизъюнкции

"|" на "+"  - знак конъюнкции

"-" на "¬" - логическое отрицание

Также имеет ограничение на вводимые символы - можно вводить лишь :

1) Обозначения логической функции - X, Y, Z

2) Знаки логических функций, скобки

3) Обозначения переменных - A, B, C, D, E, F

2)     TLogMemo – наследник TMemo - используется для вывода промежуточных результатов во время упрощения функции

3)     TLogEntry – наследник TGraphicControl – основа для логических элементов содержит объявления виртуальных базовых функций Evaluate, GetLogUr и GetCoordsIO

4)     TDragShape – наследник TLogEntry – содержит процедуры перемещения графических объектов посредством Drag-and-Drop.

5)     TLogElement – логический элемент. Содержит процедуры рисования и расчета значения логической функции, а также функции соединения посредством компонента TLogConnect.

6)     TLogConnect – наследник TClass – предназначен для хранения информации о соединениях между логическими элементами. Реализован алгоритм растягивающихся соединений

7)     TLogFuncElem – наследник TLogElement – это элемент - логическая функция

8)     TLogIn – компонент – логический вход

 

Созданная логическая схема сохраняется в файле и в дальнейшем может использоваться пользователем. При необходимости логическую схему можно выводить на печать.

Предусмотрены и обратные действия – по созданной в редакторе логической схемы программа составляет логическую функцию и таблицу истинности.

4)     Редактирование логической схемы

Также пользователь может самостоятельно собирать логическую схему, используя логические компоненты. Схему можно собирать как с самого начала, так и редактировать готовую.

·        Чтобы добавить логический элемент в схему необходимо сначала нажать левой кнопкой мыши на кнопку в окне "Элементы алгебры логики" с соответствующим обозначением элемента и затем отпустить. После этого нажмите левой кнопкой в рабочей области окна "Редактор логических схем". После этого в окне появится новый логический элемент.

·        Чтобы удалить логический элемент необходимо нажать на него левой кнопкой мыши и в выпадающем списке выбрать "Удалить"

·        Чтобы соединить логические элементы необходимо нажать на область соединения первого логического элемента, затем не отпуская переместить указатель мыши до соединителя второго элемента и отпустить.

3. Выводы: Используя эти возможности, программу можно применять для решения задач алгебры логики – упрощения логических функций, анализа логических схем и других на уроках информатики, а также факультативных занятиях, поскольку некоторые рассматриваемые вопросы выходят за рамки школьной программы.
